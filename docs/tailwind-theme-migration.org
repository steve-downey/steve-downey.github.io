* Overview

This blog moved from [[https://get.foundation/sites/docs/][ZURB Foundation 6]] to [[https://tailwindcss.com/][Tailwind CSS]].  Foundation 6's community
has gone quiet, and its global =code= / =kbd= styling fought org-mode's source
block output at every turn.  Tailwind's [[https://tailwindcss.com/docs/preflight][Preflight]] reset eliminates that conflict
structurally, and its [[https://tailwindcss.com/docs/typography-plugin][Typography plugin]] handles prose content without manual
intervention.

The theme uses the Tailwind Play CDN---no build step, no =node_modules=.  For
a personal blog the client-side CSS generation is a reasonable tradeoff.

** What Changed

- Foundation 6 replaced by Tailwind CSS (Play CDN + Typography plugin)
- All Mako templates rewritten with Tailwind utility classes
- Source blocks (=.org-src-container=) centered via =margin: auto= +
  =width: fit-content=, with language labels and copy buttons
- The =code.kbd= conflict eliminated by Tailwind's Preflight
- Syntax highlighting CSS generated by =secretaire-css.el=, which reads
  the modus-themes palette and emits CSS custom properties

** Theme Inheritance

#+begin_example
Old: sdowney-foundation6 → foundation6 → base
New: sdowney-tailwind → nikola-tailwind-blog → nikola-tailwind-base → base
#+end_example

The three-tier design separates generic Tailwind infrastructure, blog layout,
and personal customizations.  Each tier is an independent Nikola theme that
could live in its own repository.  See =building-a-tailwind-nikola-theme.org=
for the full architecture.

* The Code Block Fix

Foundation 6 globally declared:

#+begin_src css
code, kbd { background-color: #e6e6e6; }
code { border: 1px solid #cacaca; padding: .125rem .3125rem .0625rem; }
#+end_src

Org-mode's HTML exporter produces =<code>= inside =<pre class="src">= inside
=<div class="org-src-container">=.  The Foundation rules bled into these,
producing gray backgrounds on dark code blocks.  The old =custom.css= tried
to override with =code { background-color: inherit; }=, but that was fragile.

Tailwind's Preflight resets =code= and =kbd= to have no background, border, or
padding.  The Typography plugin's =.prose code= adds only color and
font-weight.  We suppress its backtick pseudo-elements in =tailwind-base.css=:

#+begin_src css
.prose :where(code)::before,
.prose :where(code)::after {
    content: none;
}
#+end_src

Defense-in-depth for source blocks ensures no framework ever conflicts:

#+begin_src css
.org-src-container pre.src code {
    background: transparent;
    border: none;
    padding: 0;
}
#+end_src

Source blocks center via =margin: auto= and size to their content via
=width: fit-content=, with a minimum of 60% to avoid excessively narrow
blocks.

* Modus Syntax Highlighting

The =modus-vivendi-tinted.css= and =modus-operandi-tinted.css= files provide
CSS classes for org-mode's htmlize output (=.org-keyword=, =.org-type=,
=.org-string=, etc.).  These are generated by =secretaire-css.el=, an Emacs
package that reads the [[https://protesilaos.com/emacs/modus-themes][modus-themes]] palette and emits CSS rules using
=var(--palette-name)= references.

The palette-aware export means that changing your modus variant automatically
updates code block colors.  The =:root= block in the exported CSS declares
=--bg-main= and =--fg-main=, which =tailwind-base.css= references via
=var(--bg-main, #1e1e2e)= with neutral fallbacks.

** Generating Syntax Highlighting CSS

#+begin_src emacs-lisp
(require 'wctm-css)
(secretaire-css-export-both-modus)
#+end_src

Or interactively: =M-x secretaire-css-menu=, then press =b= for both themes.

See =secretaire-css-design.org= for the full architecture of the palette-aware
export.

* Shadowing Nikola's Base CSS

Nikola's =base= theme ships =theme.css=, =code.css=, =rst_base.css=, and
=nikola_rst.css=.  These contain bare element selectors (=body=, =pre=,
=code=, =a:link=) that conflict with Tailwind.  The base tier shadows them
with empty files so the deployed copies are harmless.

This is preferable to removing them from the stylesheet list because Nikola's
asset system deploys base theme assets unconditionally.

* Configuration

The active theme is set in =conf.py=:

#+begin_src python
THEME = "sdowney-tailwind"
#+end_src

Font stacks, the Tailwind config, and external CDN resources are defined in
=themes/sdowney-tailwind/templates/base_helper.tmpl=.  Code block colors use
CSS custom properties defined in
=themes/nikola-tailwind-base/assets/css/tailwind-base.css=.

* For LLM Re-priming

#+begin_quote
This is a Nikola blog using Mako templates and a three-tier Tailwind CSS
theme: =nikola-tailwind-base= (framework) → =nikola-tailwind-blog= (blog
layout) → =sdowney-tailwind= (personal fonts and modus syntax highlighting).
Posts are authored in Emacs org-mode.  Source blocks use modus-vivendi syntax
highlighting via CSS classes (=.org-keyword=, =.org-type=, etc.) generated by
=secretaire-css.el=.  Code block layout is in =tailwind-base.css=.  The
Tailwind config is inline in =base_helper.tmpl=.  Build: =make compile=.
#+end_quote

Key files:
- =themes/sdowney-tailwind/templates/base_helper.tmpl= — fonts, CDN, Tailwind config
- =themes/nikola-tailwind-base/assets/css/tailwind-base.css= — code block CSS
- =themes/nikola-tailwind-blog/templates/base.tmpl= — page layout
- =plugins/orgmode/secretaire-css.el= — palette-aware CSS export
- =conf.py= — site configuration
